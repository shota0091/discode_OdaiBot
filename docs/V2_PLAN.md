# お題Bot v2.0 開発計画

## 現状 (v1.2)
- ダッシュボード、お題管理、タグ管理、スケジュール管理、設定
- ユーザー管理（BAN・ロック・招待・プロフィール）
- メモ機能、更新日時、メモ検索、フィルター・ソート
- 招待管理ページ、テスト投稿プレビュー
- パスワードリセット・管理者PW変更

## ターゲット
**雑談系Discordサーバー**（過疎回避・会話活性化目的）

---

## v2.0 実装項目

### Phase 1 — インフラ・基盤整備（販売前に必須）

#### DBバックアップ
- cronで定期的にMySQLダンプを出力
- ローカル or 外部ストレージへの退避
- 画像BLOBが大きいため早めに対応

#### お題ファイルのローカル管理化
- 現状：画像をMySQLのLONGBLOBで保存（スケール不可）
- 変更：ファイルはサーバーのローカルパスで管理、DBにはパスのみ保持
- 旧仕様で登録済みの画像も移行対象に含める

#### APIレート制限
- ログインエンドポイントへのブルートフォース対策
- 同一IPから一定時間内に大量リクエストがあった場合に遮断
- `slowapi`（FastAPI向けライブラリ）で実装

---

### Phase 2 — ビジネスモデル基盤

#### プラン管理
- guildごとにプランを管理するテーブル設計
- 格安プラン / 通常プランなどの区分
- 他機能（デフォルトお題制限など）の前提となる

#### デフォルトお題の設定
- 運営側が提供するデフォルトのお題セット（画像 or テキスト）
- 格安プランはデフォルトお題のみ使用可能
- 通常プランは独自お題のアップロードが可能

#### デフォルトお題の利用有無
- サーバー設定でデフォルトお題を使うか選択できる
- デフォルトを使いたくないサーバー向け

---

### Phase 3 — コンテンツ拡張

#### テキスト形式のお題
- 画像ではなく文章でお題を投稿する形式
- 雑談系コミュニティの会話トピックとして活用
- 例：「今日ハマっていることは？」「好きな食べ物ランキングは？」
- 画像お題より雑談系への親和性が高い

#### テキストお題のデフォルトセット
- 会話トピック100本パックなどとして提供
- デフォルトお題販売の主力コンテンツ候補

---

### Phase 4 — 運用・分析

#### 反応数の記録・可視化
- Discordの投稿に対するリアクション数・返信数を記録
- ダッシュボードで「お題ごとの反応数」「週ごとの活性度」などを表示
- 購入者が「Bot導入前後で活性化した」と実感できる根拠になる
- 過疎回避という目的に対する効果の可視化

#### ログ集積・管理者向け解析ダッシュボード
- 販売者（運営）側が複数サーバーの稼働状況を一覧で確認できる
- エラー・失敗投稿・異常な操作のログ集積
- 課金サーバーの利用状況把握

#### 投稿失敗通知
- スケジュール投稿が失敗した場合に管理者へ通知
- Discord DM or メールでアラート

---

### Phase 5 — UX改善

#### Discord OAuth対応
- Discordアカウントで直接ログインできる仕組み
- 購入者にとってID/PW登録が不要になる
- オンボーディングのハードルを下げる

---

## 見送り項目

| 項目 | 理由 |
|------|------|
| 動画形式のお題 | Discordのファイルサイズ制限（無料8MB）により実用性が低い |
| 課金システム（Stripe等） | プラン管理の設計が固まってから検討 |
| ユニットテスト整備 | 現状の規模では費用対効果が低い |

---

## 優先順位まとめ

```
Phase 1（基盤）→ Phase 2（ビジネス）→ Phase 3（コンテンツ）
                                     → Phase 4（分析）
                                     → Phase 5（UX）
```

Phase 2のプラン管理はPhase 3以降の機能制限の前提になるため、コンテンツ拡張より先に設計だけでも固めておくことを推奨。
